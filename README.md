
# Multi-Step Attack Assistant — краткая инструкция

Расширение для Burp Suite, автоматизирующее многошаговые атаки с подстановкой переменных между запросами.
Скомпилированный файл находится по пути /build/libs/extension-template-project.jar

## Быстрый старт

### 1. Сборка последовательности

1. В **Proxy History** выдели нужные запросы (Ctrl+клик).
2. ПКМ → **Extensions → Send to Multi-Step**.
3. Запросы появятся на вкладке **Multi-Step → Sequence Editor**.

### 2. Добавление переменных

1. Выдели первый шаг, нажми **Execute Step**.
2. Перейди на вкладку **Variable Manager**.
3. Автоматически предложенные cookie-переменные отметь галочкой и нажми **Add Selected to Step**.
4. Для CSRF и других параметров нажми **Add Custom...**, укажи:
    - **Name**: `csrf`
    - **Type**: `REGEX`
    - **Pattern**: `<input[^>]*name="csrf"[^>]*value="([^"]+)"`
    - **Source step #**: номер текущего шага
5. Повтори для каждого шага, откуда нужно извлечь данные.

### 3. Подстановка переменных

1. Выдели следующий шаг, нажми **Edit Body & Cookie** (или **Edit Headers**).
2. Замени статические значения на плейсхолдеры:
   ```
   
   Cookie: session={{session_1}}
   ...
   csrf={{csrf_1}}
   mfa-code={{payload}}
   ```
3. Нажми **OK** — запрос обновится.
4. Выполни шаг кнопкой **Execute Step** и проверь, что переменные извлеклись.

### 4. Запуск атаки

1. Перейди на вкладку **Attack**.
2. Нажми **Refresh** для выбора последовательности.
3. Укажи **Payload Step #** — номер шага, куда подставляется `{{payload}}`.
4. В поле **Payloads** вставь значения (по одному на строку) или загрузи файл.
5. Установи **Threads** (3–5).
6. Нажми **Start Attack**.

## Плейсхолдеры

| Плейсхолдер     | Описание                  |
| --------------- | ------------------------- |
| `{{session_1}}` | Переменная из шага 1      |
| `{{csrf_2}}`    | Переменная из шага 2      |
| `{{payload}}`   | Текущее значение нагрузки |
| {{...}}         | ...                       |

## Фильтры и сортировка

На вкладке **Attack** доступны:
- **Search payload** — поиск по значению нагрузки.
- **Status** — фильтр по HTTP-коду (200, 302 и т.д).
- **Regex in response** — фильтр по регулярному выражению в теле ответа.
- **Сортировка** — клик по заголовку любой колонки таблицы.
- **Просмотр запроса/ответа** — клик по строке таблицы.
